How to Reduce Chrome Memory Usage

5-minute read · Reviewed July 2026

Chrome's memory use can grow as tabs, extensions, and background tasks accumulate. You do not have to close important work at random: start with Chrome's built-in controls, then use more deliberate tools only where they add useful control.

1. Use Chrome's built-in Memory Saver

Open Settings → Performance and turn on Memory Saver. Chrome deactivates tabs you are not using and reloads them when you return. You can choose how aggressively it works and add sites to Always keep these sites active.

Memory Saver and Energy Saver are separate features. Memory Saver manages inactive tabs; Energy Saver reduces power use when your device is unplugged or the battery is low.

2. Sleep tabs manually with an extension

A tab manager is useful when you want to review the exact tabs first. Tab Relief uses Chrome's chrome.tabs.discard() extension API for its sleep action. Chrome removes an eligible tab from memory while keeping it visible in the tab strip, then reloads the page when you activate it again.

Tab Relief leaves active, pinned, audible, local, and protected tabs out of its normal sleep selection. This is an extra layer of user control, not a replacement for Chrome's built-in Memory Saver.

3. Save your windows as workspaces before clearing

If a project is finished for now, save its restorable web tabs as a named workspace before closing them. This reduces the number of open tabs while preserving a convenient way to reopen the set later. The memory released depends on the pages, Chrome, and the operating system, so no fixed amount can be promised.

4. Clear duplicate and leftover tabs regularly

Duplicate and leftover tabs are easy to miss. Use search or a matching rule to gather tabs by title, domain, or URL, review the matches, and close only the ones you no longer need. Review matters because closing a tab is different from putting it to sleep.

5. Audit your extensions

Open Chrome's Task Manager to see which tabs and extensions are using resources, then review anything you no longer need at chrome://extensions. Manifest V3 background service workers normally stop after a period of inactivity, so an installed extension is not necessarily consuming memory continuously. Measure before removing tools you still use.

What actually moves the needle

Use Chrome's Task Manager or your operating system's memory view before and after a change. Pages with video, complex web apps, or active background work may use far more memory than simple documents. Chrome and the operating system also decide when released memory is reflected in system totals, so compare under the same conditions instead of relying on a universal per-tab estimate.
