How to Reduce Chrome Memory Usage

5-minute read · Applies to Chrome 121 and later

Chrome is the most memory-hungry browser most people use. Each open tab, extension, and background process takes a share of RAM, and on machines with 8 GB or less that adds up fast. The good news: you don't need to close your work to get Chrome back under control.

1. Use Chrome's built-in Memory Saver

Chrome 108 and later ships with a Memory Saver (called "Energy Saver" on some builds). Go to Settings → Performance and enable it. Chrome will automatically discard inactive tabs after a period of inactivity. Discarded tabs reload when you return to them.

Limitation: you can't choose which tabs get discarded, and there's no workspace-save step before Chrome clears them.

2. Sleep tabs manually with an extension

If you want to control exactly which tabs sleep — protecting pinned tabs, sites playing audio, or important research you're not done with — a dedicated tab-sleep extension adds that layer. Tab Relief uses the same underlying chrome.tabs.discard() API as Chrome's Memory Saver, so discarded tabs reload instantly. The difference is you decide what stays awake.

3. Save your windows as workspaces before clearing

The biggest memory drain is usually a window of 20–40 research tabs you opened for a project and never cleared. Saving that window as a named workspace takes five seconds, lets you close every tab cleanly, and restores the full set when the project comes back. This alone typically frees 300–600 MB.

4. Clear duplicate and leftover tabs regularly

Duplicate tabs — the same URL open twice because you middle-clicked by accident — are easy to miss but add up. A close-matching-tabs feature lets you sweep a domain (e.g. all YouTube tabs or all Twitter tabs) in one step, with a review screen so nothing closes unexpectedly.

5. Audit your extensions

Every active Chrome extension runs a background service worker that consumes memory. Go to chrome://extensions and disable any extension you haven't used in the last month. Extensions you use rarely can be left installed but switched off; they use no memory when disabled.

What actually moves the needle

In practice: sleeping 15–20 inactive tabs typically recovers 200–500 MB. Closing an unused project window of 30 tabs can recover 600 MB or more. Removing two unused extensions might recover 50–150 MB. The exact numbers depend on which sites you have open — JavaScript-heavy pages (email, social media, video) cost far more than plain text tabs.
