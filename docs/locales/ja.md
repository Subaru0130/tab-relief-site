# Locale: Japanese

## Native Quality Status

Status: production-supported

日本語ページは英語の翻訳ではなく、日本語のLPとして読めることを優先する。特にヒーロー見出し、価格、トライアル条件、キャンセル説明は、文として自然かを必ず確認する。

## Tone

- 短く、自然で、購入前の不安を減らす言い方にする。
- 英語の二文スローガンや名詞句をそのまま日本語にしない。
- 「〜を明確化」「〜を確認可能」のような硬い説明調は避ける。
- ボタンは「何が起きるか」が分かる表現にする。

## Editorial Workflow

日本語を英語から一文ずつ翻訳しない。次の順番で書く。

1. 先に事実、読者の状況、読後にしてほしい行動だけを箇条書きにする。この段階では英語の文を残さない。
2. 英語原文を閉じ、日本語だけを見てゼロから下書きする。見出し数、段落数、文の順番を英語版に合わせない。
3. 技術用語より先に、読者が普段使う言葉で状況を説明する。API名や内部仕様は本文の流れを止めるなら参考情報へ移す。
4. 音読し、普段の会話で言わない文は意味を保ったまま書き直す。助詞が続く文、名詞を重ねた見出し、主語が分からない文は残さない。
5. 「この文は誰が、いつ、何をする話か」を各段落で確認する。答えられない段落は削るか分ける。
6. 読了時間、確認日、制作段階など、読者の判断に不要な編集用ラベルは表示しない。
7. スクリーンショットで読んだうえで、日本語を母語とする確認者の承認を得るまで公開しない。

日本語版は英語版と同じ情報量である必要はない。読者が迷わず理解できることを優先する。

## Billing Vocabulary

- トライアルは「カード登録なしで14日間試せます」を使う。
- トライアル後は「自動で有料プランに切り替わることはありません」を使う。
- 契約、支払い方法、領収書、キャンセルは「契約・支払いページ」を使う。
- 無料トライアルの条件は「メールアドレスごとに1回」を使う。

## Forbidden Literal Phrases

- 「Chromeを軽く。タブは残す。」
- 「タブは残す。」
- 「サブスク」
- 「自動課金なし」
- 「試用済み」
- 「請求ポータル」
- 「管理画面」
- 「アカウントページ」

## Marketing And Hero Copy

- ヒーローは「タブを閉じずに、Chromeを軽く。」のように、ユーザーの不安に先回りする自然な文にする。
- 英語の語順、句読点、分割数を守る必要はない。意味ではなく目的を移す。
- プレビューや数値は、価値が伝わる状態だけを見せる。0件や0MBを主役にしない。
