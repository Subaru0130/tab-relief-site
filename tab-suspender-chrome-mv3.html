Tab Suspender for Chrome in 2026: What Works After Manifest V3

6-minute read · Updated for Chrome 121+

The Great Suspender was the most popular tab-suspender extension for years. Then Chrome disabled it over malware concerns, and the landscape fractured. Now Chrome's Manifest V3 (MV3) migration has broken most remaining MV2 suspenders. This article explains what changed and what still works.

Why classic tab suspenders stopped working

Most legacy tab suspenders used a method called URL replacement: when a tab was suspended, the extension replaced the page URL with a placeholder page. This let them show a custom "sleeping" screen but meant every suspended tab held no memory of the real page — creating a separate extension page instead.

Chrome 112+ (Manifest V3) removed persistent background pages and restricted certain APIs these extensions relied on. Many simply stopped working, or lost the ability to restore tabs correctly after a browser restart.

The modern approach: chrome.tabs.discard()

Chrome now exposes chrome.tabs.discard(), the same API Chrome's own Memory Saver uses internally. Discarding a tab via this API:

  • Frees the tab's memory immediately
  • Keeps the tab visible in the tab bar with its title and favicon
  • Reloads the original page when the user clicks the tab — no placeholder page
  • Works correctly through browser restarts

Extensions built on chrome.tabs.discard() are natively compatible with MV3 and don't require workarounds.

What to look for in a 2026 tab suspender

  • Built on chrome.tabs.discard() — confirms MV3 compatibility
  • Control over what sleeps — pinned tabs, audio tabs, and sites you're actively using should never be discarded automatically
  • Workspace save — if you want to clear a project window entirely, you need a way to save and restore the full tab set, not just sleep individual tabs
  • Auto-sleep with a configurable delay — "sleep everything older than X minutes" handles the browser passively without manual effort
